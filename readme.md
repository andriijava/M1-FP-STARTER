* Програма повноцінно підтримує два незалежних алфавіти — англійський (EN) та український (UA) із  збереженням регістру літер.

*  Реалізовано обробку помилок (відсутність файлу, невалідні прапорці, нечислові ключі). Програма не падає , а виводить зрозумілі підказки для користувача.

3.Особливості проєкту та цікаві рішення:
*  Для обробки великих або від'ємних ключів використано формулу `(key % alphabetLength + alphabetLength) % alphabetLength`.
* Це гарантує отримання валідного індексу та страхує програму від падіння в рантаймі при будь-яких вхідних даних (навіть якщо ключ націло кратний довжині алфавіту, наприклад `-26`). 
*  Брут-форс: Алгоритм зламу базується на комплексному аналізі тексту: перевіряється наявність пробілів після розділових знаків, аналізуються найчастіші дволітерні сполучення, 
* а також реалізовано перевірку на "стоп-слова" (найпопулярніші сполучники та займенники) окремо для кожної мови. 
* Захист від дублювання міток: Метод `.replace()` у логіці перейменування автоматично очищує ім'я файлу від старих міток. Це запобігає багу накопичення імен типу `file[ENCRYPTED][ENCRYPTED].txt`.
